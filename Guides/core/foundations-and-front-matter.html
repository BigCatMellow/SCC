StarCraft the Miniatures Game

A rules-first study book for learning the system, checking tricky interactions, and keeping exact wording close at hand.
What this book is trying to do
Teach the rules in a cleaner order, preserve the load-bearing RAW limits and exceptions, and give the reader better entry points into the hardest parts of the game.
Built for three jobs
Study • reference • table check

Use the ledes to learn, the rule panels to verify exact logic, and the callouts to catch common failure points before they become table disputes.

Editorial stance
RAW first, teaching second

This edition is organized to be easier to read, but it is not inventing a new rules layer. Clarification exists to reveal the rule, not replace it.

Best first read
Start with Parts I–III and VIII–X

Those parts build the vocabulary, movement logic, activation flow, and attack structure that most later disputes depend on.

Keep nearby
Use Parts XI–XV when a game state gets messy

Those parts hold the highest concentration of timing windows, keyword pressure, close-combat constraints, and ruling-sequence material.

Editorial Method
Overview
Section summary

This guide follows four principles:

  • Rule function over flavor text. Introductory and thematic text has been trimmed unless it helps explain the system.
  • Specific wording preserved where it matters. Timing windows, target legality, costs, and restrictions are stated as directly as possible.
  • General rules first, exceptions second. This makes the game easier to learn without changing the outcome.
  • Source inconsistencies are flagged, not silently fixed. Where the source set appears to use conflicting terminology, this guide notes it.
Reader warning
Clarification is not a license to improvise

If a note explains a rule interaction, treat it as a reading aid tied to the source material, not as permission to ignore the actual restriction, trigger, or legality test in the underlying rule.

RAW audit note

This edition was checked against the project RAW source files in Parts 2–12. No dead numbered cross-references were found there. Most disputes came from wording, source artifacts, or summary files rather than broken section numbering.

The source files still contain non-rule artifacts such as "<< VISUAL >>" markers, "<a5 example>" placeholders, and one duplicated broken sentence in the Deploy text. Those were treated as layout debris rather than rules text.

Second-pass RAW audit note

A second focused audit then checked battlefield logic, timing windows, and keyword interactions that affect movement and engagement. No broken numbered references were found there either, but several rulings only become clear when Parts 4, 7, 8, 10, and 11 are read together.

The highest-friction cross-part combinations were cover versus visibility, engagement versus ranged targeting, Active versus Reaction timing windows, and the keyword set PLACE / INSTANT / DISPLACEMENT / FLYING / BURROWED.

I.01
Part I
Models, Units, and Bases
This section fixes the game’s physical baseline. A model is one miniature on one base, a unit is one or more models acting together, and the base — not the sculpt — governs measurement, contact, and legality.
Page role
Physical Rules Base

Use this page to lock down the game’s physical objects before later movement and combat exceptions start layering on top.

Read for
What Measures

Most later arguments disappear once base-to-base measurement replaces sculpt shape, pose, and scenic height.

1.1 Models
Rule text

A model is a single miniature on its own base. Rules interact with the base, not the sculpt above it. Pose, weapon angle, scenic height, and dramatic flight pose do not change how the rules work.

1.2 Units
Rule text

A unit is one or more models acting together as a single formation. A unit:

  • moves as one entity
  • attacks as one entity
  • is constrained by coherency
  • is either friendly or enemy depending on who controls it.
1.3 Bases
Rule text

The base is the true game piece.

Measurements are made base-to-base.

Bases may never overlap.

If a base cannot physically fit, the model cannot go there.

Scenic basing does not change size, line of sight, or positioning.

For models on flight stands, the bottom of the stand counts as the base.

1.4 Wobbly Model Handling
Rule text

If terrain prevents a model from standing flat, both players should agree where it would stand if it could be placed normally. The model is treated as occupying that agreed position.

RAW references: Parts 2.1-2.3, 4.1.

Tags, Player Roles, Keywords, and Rule Priority
This section sets the rule language the rest of the book uses. Tags identify what something is, player-role terms identify whose effect is resolving, and keywords or specific text tell you when a general rule stops applying.
Page role
Rules Language

This page defines the labels and hierarchy that tell you what an object is, whose effect is resolving, and when specific text overrides the baseline rule.

Read for
Meaning Before Action

Separate ordinary wording from formal keywords first, then check whether the rule you are reading is general or specific.

2.1 Faction Tags
Rule text

Faction tags determine army legality.

A faction card always has a race tag and often a sub-faction tag.

A unit card or tactical card may only be included if every tag on that card appears on the faction card.

The card does not need to match every tag on the faction card; it only needs all of its own tags to be present there.

2.2 Combat Tags
Rule text

Combat tags describe battlefield nature and targeting relevance. Examples include:

  • Ground
  • Flying
  • Armoured
  • Biological
  • Mechanical
  • Light
  • Psionic

Combat tags matter for:

  • weapon target restrictions
  • surge matching
  • many special abilities and keywords.
  • Important distinction: Ground as a combat tag is not the same thing as GROUND LEVEL as elevation.
2.3 Player Roles
Rule text

These terms matter constantly:

  • Active Player: the player whose turn it is to act.
  • Controlling Player: the player currently making decisions for a specific unit, model, or token.
  • Friendly / Enemy: determined by controller and team relationship.
2.4 Keywords
Rule text

Keywords are written in BOLD CAPS and always mean the same thing wherever they appear. Two global rules matter:

  • Keywords do not stack.
  • If a numeric keyword is gained twice, use only the highest value.
2.5 Rule Priority
Rule text

When a specific card, ability, mission rule, or other explicit text contradicts a general rule, the specific rule takes precedence.

That principle is one of the most important advanced-user tools in the game. When a ruling feels uncertain, ask:

  • What is the general rule?
  • Is there a more specific rule on the card, ability, mission, or keyword?
  • If yes, the specific text wins.
RAW references: Parts 2.4-2.6, 11.
RAW clarification note

A lowercase word is not automatically its keyword version. Read the glossary meaning only when the source is clearly invoking the formal keyword. Example: "instant" in the Supply Pool section is ordinary English, not the INSTANT weapon keyword.

Part II - Dice, Tests, and Modifiers
Rules as Written Guide
II.01
Part II
Dice Language
This section gives you the game’s dice grammar. It separates rolls, tests, re-rolls, modifiers, and fixed additions so later combat and ability text can be read without guesswork.
Page role
Dice Grammar

This page builds the dice vocabulary used everywhere else, so later combat steps and ability timing are not carrying hidden assumptions.

Read for
Number Needed Vs. Value Generated

Keep target-number modifiers separate from generated values like D3+1, because they affect different parts of play.

Overview
Section summary

The game uses standard six-sided dice.

D6: roll one six-sided die.

D3: roll one D6 and halve the result, rounding up.

2D6 / 3D6: roll that many dice and add the results.

A Roll-Off is always both players rolling 2D6, with ties re-rolled.

3.1 Re-Rolls
Rule text

A re-roll replaces the original result, even if the new result is worse. Unless a rule says otherwise, a re-roll applies to one die.

3.2 Tests
Rule text

A test is a roll against a target number. There are two broad types:

  • Characteristic tests: use a unit stat such as Armour or Evade.
  • Attribute tests: use a weapon or ability value such as Hit.
  • Not every die roll is a test. Some rolls only generate a value, such as a D3 result or a Charge roll.
3.3 Modifiers
Rule text

Modifiers change the target number, not the die result.

+X modifier makes the roll easier by reducing the target number by X.

-X modifier makes the roll harder by increasing the target number by X.

Global Limits:

  • A target number cannot be improved below 2+.
  • A target number cannot be worsened above 6+.
  • A natural 6 always succeeds.
  • A natural 1 always fails.
  • A null value (-) cannot be turned into a valid roll by modifiers.
3.4 Fixed Additions
Rule text

Expressions such as D3+1 generate a value. They are not target-number modifiers.

3.5 Cocked Dice
Rule text

If a die does not land flat, or the result is not clear, re-roll it.

RAW references: Part 3.
