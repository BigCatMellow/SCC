Keyword Or Ordinary Word

Always ask whether the source is invoking a formal glossary term or just using ordinary English.

Overview
Section summary

Before looking up any individual keyword, remember the three universal questions:

  • Is it a keyword, a status, or plain instruction text?
  • Does it stack?
  • Does it override a general rule?

Global answers:

  • keywords do not stack
  • numeric versions use only the highest value
  • specific keyword text overrides general rules when it directly contradicts them.

Selective RAW lens
BURROWED does not bypass Combat Phase restrictions
RAW says
“If a BURROWED unit begins the Combat Phase Engaged, it must first use Close Ranks to remove BURROWED; if it does not or cannot perform Close Ranks, it does not attack.”
Plain English

BURROWED can allow a lot of special interactions, but it does not let an engaged unit ignore the Combat Phase gate that controls when it becomes able to fight.

If a BURROWED unit starts Combat engaged, treat Close Ranks as the gate. No Close Ranks, no attack.

Keyword triage
How to read a high-friction keyword without overreading it
1 • Type

Is this keyword changing path movement, final placement, targeting, timing, or status?

2 • Override

What general rule is it actually replacing, and what general rules does it leave alone?

3 • Timing

At what exact step does it matter — declaration, movement, armour, damage, cleanup, or some trigger window?

4 • Limit

Section summary

This guide is not a full replacement for Part 11’s complete glossary, but these keyword patterns matter constantly in play:

37.1 BUFF
Rule text

BUFF improves the named characteristic. For target-number characteristics such as Armour or Hit, BUFF reduces the number needed. For value characteristics such as Speed or RoA, BUFF increases the value.

37.2 BURROWED
Rule text

BURROWED is a status with multiple effects, including:

  • gaining HIDDEN
  • size treated as 0
  • current supply treated as 0 for disengage checks
  • cannot control or contest mission markers
  • limited allowed actions while burrowed
  • may still use special abilities unless stated otherwise
  • may make an Evade roll against every attack targeting it
  • other models can move through it under the keyword’s restrictions.
  • This is a strong example of why Part 11 must still be consulted for exhaustive keyword resolution.
37.3 BLOCKING TERRAIN
Rule text

A terrain piece with effective size 1 or greater is blocking terrain for LoS purposes, but whether it actually blocks a given shot still depends on the cover rules.

37.4 ANTI-EVADE and similar modifiers
Rule text

Many weapon keywords apply during a precise step in the attack sequence. Advanced play depends on asking not just “what does it do?” but “when does it do it?”

37.5 PLACE, INSTANT, DISPLACEMENT, AND FLYING
Rule text

These four entries create a large share of table disputes. Read them this way in RAW order:

  • PLACE removes and sets the Leading Model directly. It ignores pathing, Gap Clearance, terrain restrictions, and elevation requirements, but it still must end in a legal position.
  • INSTANT is a keyword effect on weapons and attacks. It is not the same thing as the ordinary lowercase word "instant" used elsewhere in the rules.
  • DISPLACEMENT triggers if the Leading Model ends overlapping the token or model named by that rule. Resolve the overlap immediately by setting the displaced piece into base-to-base if possible, otherwise as close as possible. The source pronouns are clumsy here, so resolve the effect immediately and consistently.
  • FLYING units ignore terrain for movement, are never Engaged, cannot Charge or be Charged, do not participate in the Combat Phase, and cannot control or contest Mission Markers.
RAW references: Part 11, Part 2.6, Parts 7-8.
RAW clarification note

BURROWED permission to use Special Abilities does not override the Combat Phase restriction. If a BURROWED unit begins the Combat Phase Engaged, it must first use Close Ranks to remove BURROWED; if it does not or cannot perform Close Ranks, it does not attack.

RAW clarification note

PLACE is not path movement. It ignores pathing, Gap Clearance, terrain restrictions, and elevation requirements, but it still checks final legality. The unit may be set Within enemy Engagement Range unless the effect says otherwise, and a unit that is PLACED loses STATIONARY because the glossary treats being PLACED as a separate way of changing position.

INSTANT stops Enemy Reaction Abilities to that attack, but it does not skip the normal attack sequence once the Batch is resolving. DISPLACEMENT resolves immediately when the overlap condition is met; do not finish the rest of the move first and then displace later.

FLYING removes engagement and most terrain movement limits, but not ordinary target, range, or visibility requirements. It also ignores Full Cover only; Direct Cover and the Elevation Dead Zone still apply unless a more specific rule says otherwise.

Check yourself • Does a keyword usually replace the whole core sequence?

No. Most keywords replace a narrow part of a sequence, not the whole sequence. The safest read is to identify the exact override and then leave the rest of the normal rules in place.

Study recap
What to remember from Part XIV
In one sentence

Keywords are precision overrides, not permission to ignore the rest of the rules engine.

Most common error

Reading a keyword as if it rewrites targeting, timing, pathing, and final legality all at once when it only touches one of those.

Memory hook

Ask four things: what changes, when, how far, and what still stays normal?

Read next

Part XV, where all of these rule layers get compressed into a table procedure for resolving disputes fast.

Quick check
Two habits that prevent most keyword mistakes
Habit one

Find the narrow override first instead of assuming the keyword replaces the whole sequence.

Habit two

After finding the override, re-apply every normal legality check that the keyword never actually removed.

A Clean Method for Resolving Disputes
This section gives you a dispute-handling order that keeps the game moving. It is a reading procedure: identify the action, find the system rule, then layer in overrides, timing, and legality.
Page role
Ruling Workflow

This page is the practical reading order for settling disagreements without inventing a new subsystem.

Read for
System Before Exception

Start with the action type and the core rule, then layer in overrides, timing, and legality checks.

Overview
Section summary

When a rules dispute happens at the table, use this order:

Step 1 - Identify the action type
Rule text

Is this:

  • army building
  • deployment
  • movement
  • ranged attack
  • charge
  • close combat
  • scoring
  • ability use
  • card-resource payment
  • or keyword/status handling?
Step 2 - Find the general system rule
Rule text

Use the relevant core section first:

  • Parts 3-8 for core battlefield procedures
  • Part 9 for pre-game and list construction
  • Part 10 for ability and card timing
  • Part 11 for keyword text.
Step 3 - Check for a specific override
Rule text

Look for:

  • card text
  • mission text
  • deployment text
  • keyword text
  • special ability text.
  • Specific text overrides the general rule.
Step 4 - Check timing
Rule text

Many arguments are really timing mistakes:

  • Was the unit activated?
  • Was the trigger window missed?
  • Was the reaction already spent this activation?
  • Was the unit even eligible while in reserves?
Step 5 - Check legality constraints
Rule text

Most illegal plays fail because one of these was ignored:

  • line of sight
  • range
  • target tag mismatch
  • engagement restrictions
  • supply limit
  • coherency
  • zone of influence
  • same elevation for marker contesting.
Step 6 - Resolve and move on
Rule text

If the text still does not settle it, use the source quick reference and agree on a consistent interpretation for the current game.

RAW references: Parts 2.6, 10, 12.9.

One-Round Summary
Use this page as a compact round map after you already know the pieces. It is strongest as a memory aid, not as a first introduction to the rules.
Study move
Read each round as pressure building

Movement creates positions, Assault converts positions into threats, Combat cashes in engagement, and Scoring decides whether the round actually gained you anything.

Table move
Ask one question per phase

Movement: where can I legally end? Assault: what pressure can I create? Combat: what am I forced to resolve? Cleanup: what did I truly control?

Start of round
Rule text

resolve start-of-round effects

verify supply.

Movement Phase
Rule text

deploy, move, disengage, or hold

first to pass claims first-player marker for Assault.

Assault Phase
Rule text

run, hold, charge, or make ranged attacks

first to pass claims first-player marker for Combat.

Combat Phase
Rule text

all engaged ground units must fight

close ranks if desired

resolve close combat attacks.

Scoring & Cleanup
Rule text

determine marker control

score VP

check end of game

resolve end-of-round effects

refresh exhausted cards

determine next initiative state.

RAW references: Parts 8.1-8.10, Part 12.
Lore note

At the setting level, a StarCraft battle is usually a sequence of reconnaissance, commitment, breakthrough, and consolidation. The round structure mirrors that rhythm: movement creates the approach, assault creates contact, combat resolves the clash, and cleanup determines what was truly gained.
